# Tooluse Labs V5a Connected Prompt Candidate

## 设计目标

这版基于 V5 prompt 方案调整：保留 CLI/prompt 的整体构图，但降低通用终端符号 `>_` 的直接感，让它更像 tool routing / tool invocation。

## 改动

- 原来的 `>` 改成三个由线连接的点，保持 “输入/调用方向” 的轮廓。
- 原来的横线改成两个点，像调用后的候选输出或工具结果。
- 线条和点都使用琥珀色体系，外层继续保留青色代码括号。
- 点加深色描边，避免小尺寸下和连接线糊在一起。

## 判断

相比 V5，这版不再像普通终端图标，和 Tooluse Labs 的 agent/tool 语义更接近。相比 V4/V4a，它仍然保留了一点 CLI 调用感，适合偏开发者工具的品牌表达。

风险是右侧两个点在极小尺寸下可能被读成省略号的一部分；如果继续精修，可以尝试把右侧改成单点，让整体更简洁。
